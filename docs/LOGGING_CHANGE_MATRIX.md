# Logging Change Matrix

Pogladowa tabela do szybkiego oznaczania zmian. W kolumnie `Decyzja` wpisz np. `OK`, `ZMIEN`, albo konkret (`INFO`, `wylacz`, nowy format).

## 1) Sinki i routing

| ID | Obszar | Sink / kanal | Aktywny kiedy | Aktualnie | Decyzja |
|---|---|---|---|---|---|
| S1 | Test runtime | `stdout` | `configure_logging(...)` | poziom: `CONSOLE_LOG_LEVEL` (domyslnie `WARNING`), format czytelny | |
| S2 | Test runtime | `artifacts/<run_id>/logs/run_<run_id>_<worker>.log` | `configure_logging(...)` | poziom: `DEBUG`, JSON `serialize=True`, `enqueue=True` | |
| S3 | Tools | `stdout` | `configure_tools_logging(...)` | poziom: `CONSOLE_LOG_LEVEL` | |
| S4 | Tools | `tools/logs/<script>.log` | `configure_tools_logging(...)` | poziom: `TOOLS_FILE_LOG_LEVEL` (domyslnie `DEBUG`) | |
| S5 | Remote logging | Reporting API sink | `add_reporting_api_sink(...)` gdy reporting wlaczony | poziom: `REPORTING_API_LOG_LEVEL` (domyslnie `WARNING`), endpoint: `/test-run/log` | |
| S6 | Bridge | stdlib `logging` -> Loguru | zawsze po konfiguracji loggera | `_InterceptHandler` + `logging.captureWarnings(True)` | |

## 2) Opcje konfiguracyjne (settings/env)

| ID | Klucz | Typ | Aktualna wartosc domyslna | Uzywane przez | Decyzja |
|---|---|---|---|---|---|
| C1 | `console_log_level` / `CONSOLE_LOG_LEVEL` | level | `WARNING` | sinki konsolowe | |
| C2 | `tools_logs_dir` | path | `tools/logs` | sink plikowy tools | |
| C3 | `tools_file_log_level` / `TOOLS_FILE_LOG_LEVEL` | level | `DEBUG` | sink plikowy tools | |
| C4 | `log_rotation` / `LOG_ROTATION` | string | `50 MB` | sink run JSON + tools file | |
| C5 | `log_retention` / `LOG_RETENTION` | string | `7 days` | sink run JSON + tools file | |
| C6 | `log_compression` / `LOG_COMPRESSION` | string | `zip` | sink run JSON + tools file | |
| C7 | `reporting_api_log_endpoint` / `REPORTING_API_LOG_ENDPOINT` | path | `/test-run/log` | sink API logow | |
| C8 | `reporting_api_log_level` / `REPORTING_API_LOG_LEVEL` | level | `WARNING` | filtr sinka API | |

## 4) Format i payload

| ID | Obszar | Aktualnie | Opcje zmiany | Decyzja |
|---|---|---|---|---|
| F1 | Console test | format z metadanymi (`run_id`, `worker`, `browser`, `tester`, `note`, `nodeid`) | skrocic / rozszerzyc | |
| F2 | Run file | JSON `serialize=True`, `backtrace=True`, `diagnose=True` | zmiana verbosity traceback | |
| F3 | Tools file | tekst `time|level|name:function:line|message` | dodac `run_id`/`worker` | |
| F4 | API log payload | `event_type=log`, `timestamp`, `level`, `message`, `run_id`, `nodeid`, `metadata`, `extra` | dodac/usunac pola | |
| F5 | Redaction | maskowanie secretow w `message` i `extra` | rozszerzyc wzorce / bez zmian | |

## 5) Przelaczniki operacyjne (quick picks)

| ID | Scenariusz | Ustawienia startowe | Decyzja |
|---|---|---|---|
| P1 | Dzienny run CI (malo szumu) | `CONSOLE_LOG_LEVEL=WARNING`, `REPORTING_API_LOG_LEVEL=WARNING` | |
| P2 | Diagnostyka CI | `CONSOLE_LOG_LEVEL=INFO`, `REPORTING_API_LOG_LEVEL=INFO` | |
| P3 | Incident / deep debug | `CONSOLE_LOG_LEVEL=DEBUG`, `TOOLS_FILE_LOG_LEVEL=DEBUG`, `REPORTING_API_LOG_LEVEL=WARNING` | |
